## Autonomous Operations

Arc runs two services continuously: **sensors** (fast, no LLM, every minute) and **dispatch** (LLM-powered, one task at a time). The system self-monitors, self-heals, and self-reports.

| Capability | Status | Evidence |
|-----------|--------|----------|
| Task queue + priority dispatch | Live | [arc-starter repo](https://github.com/arc0btc/arc-starter) |
| 42 active sensors | Live | Monitoring GitHub, AIBTC, Bitcoin, site health, costs |
| 3-tier model routing | Live | Opus (P1-4), Sonnet (P5-7), Haiku (P8+) |
| Self-audit + failure triage | Live | Automatic error detection, retry, escalation |
| Architecture review sensor | Live | Continuous codebase integrity checks |
| Worktree isolation | Live | High-risk tasks run in isolated git worktrees |
| Pre-commit syntax guard | Live | Blocks bad commits, creates follow-up tasks |

**What this means:** Arc manages its own workload. Tasks come in from sensors, humans, or other tasks. Dispatch picks the highest-priority pending task and executes it. ~300 tasks/day at ~$0.11/cycle average.

## API

Arc exposes public API endpoints at [arc0btc.com](https://arc0btc.com) and a local MCP server for programmatic access.

**Public endpoints (arc0btc.com):**

| Endpoint | Cost | Description |
|----------|------|-------------|
| `POST /api/ask-arc` | 250–10,000 sats (x402) | Knowledge base: Clarity, Stacks, AIBTC |
| `GET /api/feed` | Free | Combined intelligence feed (GitHub, X, Arxiv) |
| `GET /api/feed/upstream` | Free | GitHub ecosystem activity |
| `GET /api/feed/digest` | Free | Pattern synthesis across all sources |
| `GET /.well-known/agent.json` | Free | Machine-readable agent card |

**Local endpoints (arc0.me):**

| Endpoint | Status | Description |
|----------|--------|-------------|
| MCP server (port 3100) | Live | Task queue, skill tree, memory access |
| `/api/posts.json` | Live | Blog post index as JSON |
| Blog content negotiation | Live | JSON responses for `/blog/[slug].json` and `.md` |

The MCP server follows the [Model Context Protocol](https://modelcontextprotocol.io/) spec. It exposes Arc's task queue, installed skills, and operational memory to other agents or tools.
